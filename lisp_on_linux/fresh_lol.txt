How To Make A Fresh Redlinux Install: (32 or 64 bits)

1. Install the latest version of ArchLinux whether 32 or 64 bits.
2. Login as root and run the following commands:
wget http://redlinernotes.com/docs/lisp_on_linux/pkglist.txt
pacman -S `cat pkglist.txt`
3. Add new user, enable wheel group in sudoers. Login as new user.
4. Extract and insert default config files:
wget http://redlinernotes.com/docs/lisp_on_linux/configs.tar.gz
tar -zxvf configs.tar.gz
mv configs/* ~/
sudo mv ~/etc/* /etc/*
5. mkdir ~/builds and git clone git://gitorious.org/~redline6561/clbuild/clbuild-dev.git. In clbuild...
./clbuild update --all-projects
cd source/paktahn-dev/ && sbcl
(require 'paktahn)
(in-package :pak)
(build-core)
;;; ALTERNATELY: Install paktahn binary using lpolzer's quickinstall script.
6. Add .paktahn/config.lisp with "(in-package :pak) (setf *color-scheme* *color-scheme-lightbg*)".
7. Run the following commands:
cd ~
wget http://redlinernotes.com/docs/redlinux/assets.tar.gz
tar -zxvf assets.tar.gz
mv wallpapers images/wallpapers
mv Crashbit .icons/Crashbit
wget http://redlinernotes.com/docs/lisp_on_linux/aurlist.txt
paktahn -S `cat aurlist.txt`
8. Add halt, reboot and pm-suspend to /etc/sudoers.
9. Go to ~/builds/clbuild/shuffletron, edit Makefile so PREFIX=/home/redline/builds and make, make install
10. Hand-install any necessary larch stuff...and LARCH IT. Note: Larch only supported on 32 bit systems at this time.
11. Test it, document stuff, test it, document stuff, larch it, update the changelog, release it! Files with guest in them: .asdf-install, .sbclrc, .eclrc, mpd.conf, etc.
